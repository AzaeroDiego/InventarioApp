🚀 PASOS PARA COMENZAR
═════════════════════════════════════════════════════════════════════════════

PASO 1: Instalar Python
┌─────────────────────────────────────────────────┐
│ 1. Descarga Python 3.9+ desde python.org       │
│ 2. Instala con "Add Python to PATH" marcado   │
│ 3. Verifica: python --version                  │
│ 4. Verifica: pip --version                     │
└─────────────────────────────────────────────────┘

PASO 2: Abrir carpeta en terminal
┌─────────────────────────────────────────────────┐
│ cd C:\Users\azaer\Documents\Diego\Bering\InventarioApp
│                                                  │
│ O en PowerShell:                                │
│ cd 'C:\Users\azaer\Documents\Diego\Bering\InventarioApp'
└─────────────────────────────────────────────────┘

PASO 3: Crear virtual environment
┌─────────────────────────────────────────────────┐
│ python -m venv venv                            │
│                                                  │
│ Espera a que termine (puede tomar 30 seg)     │
└─────────────────────────────────────────────────┘

PASO 4: Activar virtual environment
┌─────────────────────────────────────────────────┐
│ En PowerShell:                                  │
│ .\venv\Scripts\Activate.ps1 (o .ps1 según ver)│
│                                                  │
│ En CMD:                                         │
│ venv\Scripts\activate                           │
│                                                  │
│ Verás: (venv) en la línea de comandos          │
└─────────────────────────────────────────────────┘

PASO 5: Instalar dependencias
┌─────────────────────────────────────────────────┐
│ pip install -r requirements.txt                │
│                                                  │
│ Espera a que termine (2-3 minutos)             │
│ Verás: "Successfully installed..."             │
└─────────────────────────────────────────────────┘

PASO 6: Inicializar base de datos
┌─────────────────────────────────────────────────┐
│ flask init-db                                  │
│                                                  │
│ Verás: "BD inicializada satisfactoriamente"   │
│ Se crean: inventario.db y qr_codes/ folder    │
└─────────────────────────────────────────────────┘

PASO 7: Ejecutar aplicación
┌─────────────────────────────────────────────────┐
│ python run.py                                  │
│                                                  │
│ Verás: "Running on http://127.0.0.1:5000"    │
│ Presiona Ctrl+C para detener                   │
└─────────────────────────────────────────────────┘

🐛 SOLUCIÓN DE PROBLEMAS COMUNES
═════════════════════════════════════════════════════════════════════════════

PROBLEMA: "python no es reconocido"
┌─────────────┬────────────────────────────────────┐
│ Causa       │ Python no en PATH                  │
│ Solución    │ 1. Reinstala Python                │
│             │ 2. Marca "Add Python to PATH"     │
│             │ 3. Reinicia terminal               │
└─────────────┴────────────────────────────────────┘

PROBLEMA: "No módulo Flask"
┌─────────────┬────────────────────────────────────┐
│ Causa       │ Dependencias no instaladas         │
│ Solución    │ pip install -r requirements.txt    │
└─────────────┴────────────────────────────────────┘

PROBLEMA: "flask: command not found"
┌─────────────┬────────────────────────────────────┐
│ Causa       │ Venv no activado                   │
│ Solución    │ .\venv\Scripts\activate (PowerShell)
│             │ venv\Scripts\activate (CMD)        │
└─────────────┴────────────────────────────────────┘

PROBLEMA: "Puerto 5000 en uso"
┌─────────────┬────────────────────────────────────┐
│ Causa       │ Otra app en puerto 5000            │
│ Solución    │ 1. Cierra otra ventana             │
│             │ 2. Espera 1 minuto                 │
│             │ 3. Cambia: export FLASK_RUN_PORT=5001
└─────────────┴────────────────────────────────────┘

PROBLEMA: "inventario.db error"
┌─────────────┬────────────────────────────────────┐
│ Causa       │ BD corrompida o permisos           │
│ Solución    │ 1. Borra inventario.db             │
│             │ 2. flask init-db nuevamente        │
└─────────────┴────────────────────────────────────┘

PROBLEMA: "Login no funciona"
┌─────────────┬────────────────────────────────────┐
│ Causa       │ BD no inicializada                 │
│ Solución    │ 1. Verifica inventario.db          │
│             │ 2. flask init-db                   │
│             │ 3. Usa demo@example.com / 123456   │
└─────────────┴────────────────────────────────────┘
