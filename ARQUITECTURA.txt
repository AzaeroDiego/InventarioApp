💾 BASE DE DATOS (6 TABLAS)
═════════════════════════════════════════════════════════════════════════════

USUARIOS (Autenticación)
├─ id (PK)
├─ nombre_completo
├─ email (UNIQUE)
├─ password_hash
├─ activo
└─ fecha_creacion

CATEGORÍAS (Clasificación)
├─ id (PK)
├─ nombre (UNIQUE)
├─ color (hex)
├─ descripcion
├─ activa
└─ fecha_creacion

PRODUCTOS (Inventario)
├─ id (PK)
├─ nombre
├─ sku (UNIQUE)
├─ categoria_id (FK → CATEGORÍAS)
├─ precio
├─ cantidad_stock
├─ stock_minimo
├─ imagen_url
├─ codigo_qr
├─ activo
├─ fecha_creacion
└─ fecha_actualizacion

MOVIMIENTOS_STOCK (Auditoría)
├─ id (PK)
├─ producto_id (FK → PRODUCTOS)
├─ usuario_id (FK → USUARIOS)
├─ tipo (entrada/salida/ajuste/devolucion/correccion)
├─ cantidad
├─ motivo (Compra/Venta/Ajuste/Devolucion/Otros)
├─ observaciones
└─ fecha

ALERTAS_STOCK (Notificaciones)
├─ id (PK)
├─ producto_id (FK → PRODUCTOS)
├─ tipo (bajo_stock)
├─ mensaje
├─ leida
└─ fecha_creacion

ESTADÍSTICAS (Reportes)
└─ Calculadas en tiempo real desde otras tablas

📊 FLUJO DE DATOS
═════════════════════════════════════════════════════════════════════════════

EJEMPLO: Usuario intenta registrarse

1. Usuario llena formulario en /auth/registro
   └─> HTML form → POST /auth/registro

2. Flask recibe en auth.py
   └─> Valida email, password
   └─> Hashea contraseña
   └─> Crea registro USUARIOS

3. Datos en BD
   └─> INSERT INTO usuarios (nombre, email, password_hash)
   └─> SQLITE procesa y guarda

4. Respuesta a usuario
   └─> Redirect a login
   └─> Flash message "Registrado éxitosamente"

5. Usuario puede loguearse
   └─> Valida credenciales
   └─> Crea sesión HttpOnly
   └─> Redirige a dashboard

🔄 CICLO DE VIDA DE UN PRODUCTO
═════════════════════════════════════════════════════════════════════════════

1. CREACIÓN
   USER → Formulario crear_producto.html
        ↓
   FLASK → main.py/crear_producto()
        ↓
   VALIDACIÓN → Nombre, SKU, categoría, precio
        ↓
   IMAGEN → Sube, valida extensión, genera UUID
        ↓
   QR → Genera código QR automático
        ↓
   BD → INSERT INTO productos
        ↓
   RESPUESTA → Redirect dashboard + Flash

2. EDICIÓN
   USER → Click editar en productos.html
        ↓
   FLASK → GET editar_producto.html con datos
        ↓
   USUARIO → Modifica campos + enviá
        ↓
   VALIDACIÓN → Verifica cambios
        ↓
   BD → UPDATE productos
        ↓
   RESPUESTA → Redirect a lista

3. MOVIMIENTOS DE STOCK
   USER → Entrada/Salida en sidebar
        ↓
   FLASK → Formulario entrada_stock.html
        ↓
   USUARIO → Selecciona producto, cantidad, motivo
        ↓
   JAVASCRIPT → Valida stock disponible
        ↓
   VALIDACIÓN → No permite overselling
        ↓
   BD → INSERT movimientos_stock
        ↓
   UPDATE → producto.cantidad_stock = nueva_cantidad
        ↓
   ALERTAS → Si stock < minimo → INSERT alertas_stock
        ↓
   RESPUESTA → Redirect con confirmación

4. REPORTE
   USER → Click en "Reportes"
        ↓
   FLASK → GET /main/reportes
        ↓
   BD → SELECT agregaciones últimos 30 días
        ↓
   JAVASCRIPT → Chart.js dibuja gráficos
        ↓
   USUARIO → Ve gráficos interactivos
        ↓
   USUARIO → Click "Exportar CSV"
        ↓
   FLASK → api.py/exportar_csv()
        ↓
   PANDAS → Convierte datos a CSV
        ↓
   RESPUESTA → Content-Disposition: attachment
        ↓
   NAVEGADOR → Descarga archivo


🔐 FLUJO DE AUTENTICACIÓN
═════════════════════════════════════════════════════════════════════════════

LOGIN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Usuario ingresa email/password
           ↓
    Valida formato email
           ↓
    Busca usuario en BD
           ↓
    Compara password:
    - Extrae hash de BD
    - Hash de entrada
    - werkzeug.check_password_hash()
           ↓
    Si OK: Crea sesión
    Si NO: Muestra error
           ↓
    Sesión HttpOnly (30 días)
    ↓
    Flask-Login tracked


PROTECCIÓN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Cada ruta usa @login_required
           ↓
    Si no autenticado: Redirect login
           ↓
    Si autenticado: Permite acceso
           ↓
    CSRF token en forms
           ↓
    Valida tokens form


LOGOUT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Usuario click logout
           ↓
    Flask-Login limpia sesión
           ↓
    Cookie se elimina
           ↓
    Redirect login

📈 CAPACIDAD DEL SISTEMA
═════════════════════════════════════════════════════════════════════════════

Base de Datos:
├─ SQLite soporta: ~1,000,000 registros fácilmente → Escalable a millones
├─ Movimientos: 1,000 diarios = ~30,000 mensuales → OK para pymes
├─ Tamaño BD: Inicio 1MB → ~500MB en 5 años de datos

Usuarios:
├─ Concurrent: 10-20 usuarios simultáneos (SQLite)
├─ Total: Ilimitados (histórico)
└─ Roles: Actualizable a admin/staff/viewer

Productos:
├─ Máximo: 10,000 sin problemas
├─ QR: Uno por producto, almacenado en disco
├─ Imágenes: 16MB máximo por upload

Rendimiento:
├─ Dashboard: < 1seg
├─ Búsqueda: < 500ms (hasta 5,000 productos)
├─ Reportes gráficos: < 2seg
└─ CSV export: < 5seg (10,000 registros)

Almacenamiento:
├─ BD: 5MB (arranque) → 500MB (5 años)
├─ Imágenes: 50MB (100 productos × 500KB)
├─ QR codes: 10MB (2,000 códigos × 5KB)
└─ Total: ~600MB máximo
